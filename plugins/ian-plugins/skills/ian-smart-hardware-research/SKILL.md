# 智能硬件产品用户调研分析

## 使用方式

**本技能仅通过显式调用触发**。

正确调用方式：
- "使用 ian-smart-hardware-research 调研智能手表用户反馈"
- "用智能硬件调研技能分析扫地机器人用户需求"
- "调用用户调研技能，研究智能药盒的市场痛点"

## 技能概述

本技能通过系统化搜索和分析社交媒体、电商平台的用户讨论，为智能硬件产品提供全面的用户需求洞察。帮助产品经理、市场人员快速了解目标用户的声音，发现产品机会。

## 输出要求

### 必交产物
1. **Markdown 报告** - 调研底稿，包含原始数据和分析结论
2. **HTML 报告** - 展示成品，使用 `brutally minimal` 风格
3. **数据校验** - 必须调用 `fact-checker` 技能验证关键数据

### 输出流程
```
目标确认 → 多平台搜索 → 数据整理 → Markdown报告 → fact-checker校验 → HTML成品
```

## 适用平台

- **知乎**：深度讨论、使用体验分享、专业评测
- **小红书**：生活方式、使用场景、产品推荐
- **微博**：热点话题、争议讨论、用户反馈
- **什么值得买**：购买决策、性价比分析、使用评价
- **京东/淘宝评价**：真实用户反馈、产品优缺点

## 调研维度

| 维度 | 说明 |
|------|------|
| 核心需求 | 用户购买和使用的主要目的 |
| 使用场景 | 具体在什么情境下使用该产品 |
| 痛点问题 | 使用过程中遇到的困难和不满 |
| 讨论热点 | 用户讨论最多的话题和争议点 |
| 好评亮点 | 用户最满意的功能或特性 |
| 决策因素 | 影响购买决策的关键因素 |

## 工作流程

### Phase 1：明确调研目标

确认以下信息：
- 产品名称（如：智能药盒、智能手表、智能家居摄像头）
- 目标用户群体（可选，如：老年人、宝妈、运动爱好者）
- 具体关注点（可选，如：续航、易用性、价格）

### Phase 2：多平台搜索

针对每个平台执行搜索，收集用户讨论：

搜索关键词组合：
- "{产品名称} 使用体验"
- "{产品名称} 评测"
- "{产品名称} 怎么样"
- "{产品名称} 推荐/避坑"
- "{产品名称} 问题/缺点"

**工具使用：**
- 中文搜索：`mcp__zhipu-web-search__web-search`
- 英文搜索：`mcp__MiniMax__web_search`
- 网页阅读：`WebFetch` / `WebReader`

### Phase 3：数据整理与分析

将收集到的信息按以下维度分类：

1. **需求分析**：提炼用户的核心需求和使用场景
2. **痛点挖掘**：识别高频出现的问题和抱怨
3. **热点识别**：总结讨论最激烈的话题
4. **好评整理**：归纳用户认可的卖点

数据质量要求：
- 同一观点在不同平台多次出现才视为有效
- 注意区分真实用户、测评博主、水军的观点
- 优先参考最近6-12个月的讨论
- 每个平台至少收集20+条有效反馈

### Phase 5：数据校验

**必须调用 `fact-checker` 技能**，校验重点：
- [ ] 用户评价的真实性（排除水军）
- [ ] 关键数据的时效性
- [ ] 样本量是否充足
- [ ] 观点是否有代表性

### Phase 6：HTML 报告生成

**调用 `frontend-design` 技能**：

## 质量控制清单

```
□ 调研目标明确（产品、用户群体、关注点）
□ 多平台搜索完成（至少3个平台）
□ 样本量充足（每平台20+条有效反馈）
□ 数据按维度分类完整
□ Markdown 报告结构完整
□ fact-checker 已调用并修正错误
□ HTML 报告使用 brutally minimal 风格
□ 数据局限性已说明
□ 报告日期已标注
```

## 注意事项

1. **显式调用** - 本技能仅响应显式调用，不响应关键词
2. **数据交叉验证** - 同一观点在不同平台多次出现才视为有效
3. **区分用户类型** - 注意区分真实用户、测评博主、水军的观点
4. **时效性** - 优先参考最近6-12个月的讨论
5. **样本量** - 每个平台至少收集20+条有效反馈
6. **客观中立** - 如实呈现正负两面的用户声音
7. **必须校验** - 所有关键数据必须经过 fact-checker 验证
