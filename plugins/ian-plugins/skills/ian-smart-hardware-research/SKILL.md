# 智能硬件产品用户调研分析

## 概述

本技能通过系统化搜索和分析社交媒体、电商平台的用户讨论，为智能硬件产品提供全面的用户需求洞察。帮助产品经理、市场人员快速了解目标用户的声音，发现产品机会。

## 适用平台

- **知乎**：深度讨论、使用体验分享、专业评测
- **小红书**：生活方式、使用场景、产品推荐
- **微博**：热点话题、争议讨论、用户反馈
- **什么值得买**：购买决策、性价比分析、使用评价
- **京东/淘宝评价**：真实用户反馈、产品优缺点

## 调研维度

| 维度 | 说明 |
|------|------|
| 核心需求 | 用户购买和使用的主要目的 |
| 使用场景 | 具体在什么情境下使用该产品 |
| 痛点问题 | 使用过程中遇到的困难和不满 |
| 讨论热点 | 用户讨论最多的话题和争议点 |
| 好评亮点 | 用户最满意的功能或特性 |
| 决策因素 | 影响购买决策的关键因素 |

## 工作流程

### 步骤 1：明确调研目标

确认以下信息：
- 产品名称（如：智能药盒、智能手表、智能家居摄像头）
- 目标用户群体（可选，如：老年人、宝妈、运动爱好者）
- 具体关注点（可选，如：续航、易用性、价格）

### 步骤 2：多平台搜索

针对每个平台执行搜索，收集用户讨论：

搜索关键词组合：
- "{产品名称} 使用体验"
- "{产品名称} 评测"
- "{产品名称} 怎么样"
- "{产品名称} 推荐/避坑"
- "{产品名称} 问题/缺点"

### 步骤 3：数据整理与分析

将收集到的信息按以下维度分类：

1. **需求分析**：提炼用户的核心需求和使用场景
2. **痛点挖掘**：识别高频出现的问题和抱怨
3. **热点识别**：总结讨论最激烈的话题
4. **好评整理**：归纳用户认可的卖点

### 步骤 4：输出结构化报告

生成包含以下内容的报告：

1. **执行摘要** - 核心发现概述
2. **详细分析表格** - 按维度整理的用户反馈
3. **洞察与建议** - 基于数据的产品建议

## 注意事项

1. **数据交叉验证**：同一观点在不同平台多次出现才视为有效
2. **区分用户类型**：注意区分真实用户、测评博主、水军的观点
3. **时效性**：优先参考最近6-12个月的讨论
4. **样本量**：每个平台至少收集20+条有效反馈
5. **客观中立**：如实呈现正负两面的用户声音
