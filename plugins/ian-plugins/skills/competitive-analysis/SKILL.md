# 消费品竞品分析工作流

## 触发指令

在新会话中，使用以下任一方式触发本工作流：

```
"我要进行[产品品类]的竞品分析"
"分析[产品品类]市场竞争"
"调研[产品品类]竞品"
"帮我做[产品品类]的市场调研"
```

示例：
- "我要进行咖啡磨豆机的竞品分析"
- "分析智能手表市场竞争"
- "调研无线耳机竞品"

## Phase 1: 信息收集

### 1.1 品牌识别
- [ ] 搜索行业头部品牌（3-5个）
- [ ] 识别每个品牌的产品线
- [ ] 记录各品牌市场定位

**工具使用：**
```
WebSearch: "[品类]品牌排行榜"
WebSearch: "[品类]知名品牌推荐"
```

### 1.2 产品参数收集

| 收集维度 | 关键字段 | 重要性 |
|---------|---------|-------|
| 基础规格 | 精度、量程、尺寸、重量 | ⭐⭐⭐ |
| 连接性能 | 蓝牙版本、App支持 | ⭐⭐⭐ |
| 续航能力 | 电池容量、续航时间 | ⭐⭐⭐ |
| 防护性能 | 防水等级、防尘等级 | ⭐⭐ |
| 价格信息 | 官方价、活动价、渠道价 | ⭐⭐⭐⭐⭐ |
| 用户评价 | 好评率、差评点 | ⭐⭐⭐ |

**工具使用：**
```
WebSearch: "[品牌] [产品] 参数 规格"
WebSearch: "[品牌] [产品] 价格"
```

### 1.3 用户评价采集
- [ ] 电商平台评价（京东、天猫）
- [ ] 专业测评文章
- [ ] 社交媒体反馈（知乎、小红书）

## Phase 2: 分析框架

### 2.1 参数对比矩阵
- 横向：各品牌产品
- 纵向：技术参数
- 标注：优势项高亮显示（🟢绿色）

### 2.2 功能对比矩阵

| 符号 | 含义 |
|-----|------|
| ✓ | 完全支持 |
| ✗ | 不支持 |
| △ | 部分支持/需配合 |

### 2.3 市场定位图
- **X轴**：专业级 → 消费级
- **Y轴**：低价格 → 高价格
- 标注各品牌位置及市场空白点

### 2.4 核心分析模型

#### SWOT分析
- **S** - 优势（Strengths）
- **W** - 劣势（Weaknesses）
- **O** - 机会（Opportunities）
- **T** - 威胁（Threats）

#### Porter五力分析
1. 新进入者威胁
2. 供应商议价能力
3. 买方议价能力
4. 替代品威胁
5. 行业内竞争

## Phase 3: 输出模板

### 3.1 报告结构

```
1. 执行摘要（核心发现）
2. 品牌详细分析（每品牌单独章节）
   - 品牌定位
   - 产品线逐个分析
   - 核心参数表
   - 优缺点分析
3. 竞品对比汇总
   - 参数对比表
   - 功能对比矩阵
   - 市场定位图
4. 核心亮点分析
5. 适用场景推荐
6. 结论与建议
```

### 3.2 输出格式

| 格式 | 用途 | 命令 |
|-----|------|------|
| Markdown | 便于编辑 | Write .md |
| HTML | 便于展示 | 使用 Skill: frontend-design |

## Phase 4: 质量控制

### 4.1 数据验证
- [ ] 交叉验证价格信息（多平台对比）
- [ ] 核实技术参数来源
- [ ] 标注数据置信度（已验证/未验证）

### 4.2 免责声明
必须包含：
- 数据来源标注
- 验证状态说明
- 决策风险提示

## 标准检查清单

```
□ 品牌覆盖全面（3-5个主要品牌）
□ 产品数量充足（每品牌2-4款产品）
□ 参数对比详细（10+项技术参数）
□ 功能矩阵完整（15+项功能点）
□ 价格信息准确（已验证）
□ 适用场景明确（6类用户推荐）
□ 市场机会清晰（空白点识别）
□ 格式美观专业（HTML可视化）
```

## 注意事项

1. **数据时效性** - 价格和技术参数会变动，需标注调研时间
2. **来源可靠性** - 优先使用官方数据和权威测评
3. **客观中立** - 避免主观偏见，用事实说话
4. **用户视角** - 关注用户真实需求和使用场景
5. **商业敏感** - 注意商业机密和竞对敏感度
