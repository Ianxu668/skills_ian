# 竞品分析报告生成

## 技能说明

本技能用于对消费品进行系统性竞品分析，生成专业的 Markdown 和 HTML 对比报告。

## 触发方式

使用 `/竞品分析` 命令触发，或通过关键词：
- "竞品分析"
- "产品调研"
- "市场调研"
- "品牌对比"

## 工具使用原则

1. **NotebookLM 链接**：优先调用 NotebookLM skill 获取内容
2. **其他链接**：使用 WebFetch 获取
3. **中文搜索**：使用 `mcp__zhipu-web-search__web-search`
4. **英文搜索**：使用 `mcp__MiniMax__web_search`

## 工作流程（5个阶段）

### 阶段 1：产品汇总

**任务**：解析用户输入，汇总所有要分析的产品

**执行步骤**：
1. 读取用户提供的本地文档（如有）
2. 解析用户提供的链接资料
3. 提取所有目标产品和竞品品牌
4. 汇总成产品列表

**输出**：明确列出所有待分析的产品名称和品牌

---

### 阶段 2：单独产品调研

**任务**：针对每个产品建立独立的 Markdown 文档资料

**执行步骤**：
1. 对每个产品进行独立搜索
2. 优先使用用户提供的文档和链接
3. 提取关键参数：价格、技术规格、功能特性
4. 创建独立的 Markdown 文件作为数据源

**输出格式**（每个产品一个文件）：
```markdown
# [产品名称] 调研资料

## 基本信息
- 品牌：
- 价格：
- 上市时间：

## 技术参数
[详细参数表格]

## 功能特性
[列表]

## 数据来源
- 来源1
- 来源2
```

---

### 阶段 3：生成汇总报告

**任务**：整合所有资料，生成最终对比报告

**执行步骤**：
1. 汇总阶段 2 的所有产品数据
2. 按照参考格式组织内容
3. 生成对比表格和分析

**报告格式参考**（见 references/report-template.md）：
- 产品线概览表格
- 详细参数对比表
- 技术创新详解
- 产品定位分析
- 选择建议

---

### 阶段 4：数据校验

**任务**：验证所有数据的准确性和来源

**执行步骤**：
1. 调用 `fact-checker` 技能校验关键数据
2. 标注数据来源和可信度
3. 修正错误信息

**校验重点**：
- 价格信息
- 技术参数
- 发布时间

---

### 阶段 5：HTML 报告生成

**任务**：生成可视化的 HTML 报告

**执行步骤**：
1. 调用 `frontend-design` 技能
2. 指定风格：`brutally minimal`
3. 生成 HTML 格式报告

---

## 输出要求

### Markdown 报告结构

```markdown
# [产品品类] 竞品对比分析报告

## 1. 产品线概览
[品牌/型号/价格区间表格]

## 2. 详细参数对比
[核心技术参数表格]

## 3. 技术创新详解
[各产品技术特点分析]

## 4. 产品定位分析
[用户画像/选择建议]

## 5. 规格总结表
[特性对比矩阵]

*数据来源: [来源列表]*
*报告生成日期: YYYY年MM月*
```

### HTML 报告

- 风格：brutally minimal
- 包含：参数表格、技术对比图、选择建议
