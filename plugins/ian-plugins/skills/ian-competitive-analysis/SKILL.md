# 消费品竞品分析

## 使用方式

**本技能仅通过显式调用触发**，不再响应关键词。

正确调用方式：
- "使用 ian-competitive-analysis 技能分析咖啡机"
- "用竞品分析技能调研智能手表"
- "调用竞品分析技能，分析无线耳机市场"

## 技能概述

本技能提供两种分析模式，根据用户需求自动选择或手动指定：

| 模式 | 适用场景 | 输出重点 |
|------|---------|---------|
| **单品深度分析** | 针对1-2个产品做详细拆解 | 技术参数、功能特性、用户评价 |
| **多品牌对比** | 分析3-5个品牌的竞争格局 | 市场定位、参数对比、SWOT分析 |

## 输出要求

### 必交产物
1. **Markdown 报告** - 技术底稿，包含完整数据和分析
2. **HTML 报告** - 展示成品，使用 `brutally minimal` 风格
3. **数据校验** - 必须调用 `fact-checker` 技能验证关键数据

### 输出流程
```
数据收集 → Markdown草稿 → fact-checker校验 → HTML成品
```

## 工作流程

### Phase 1: 模式选择

确认用户需求：
- "我要深入分析XX产品" → 单品深度分析模式
- "我要对比多个品牌的XX" → 多品牌对比模式

### Phase 2: 数据收集

#### 单品深度分析
- [ ] 搜索产品官方资料
- [ ] 收集技术参数和规格
- [ ] 采集用户评价（电商、社交媒体）
- [ ] 整理价格信息

#### 多品牌对比
- [ ] 识别行业头部品牌（3-5个）
- [ ] 收集每个品牌的产品线
- [ ] 建立参数对比矩阵
- [ ] 分析市场定位

### Phase 4: 数据校验

**必须调用 `fact-checker` 技能**，校验重点：
- [ ] 价格信息（多平台交叉验证）
- [ ] 技术参数（与官方资料核对）
- [ ] 发布时间（验证时效性）
- [ ] 用户评价（样本量和代表性）

### Phase 5: HTML 报告生成

**调用 `frontend-design` 技能**，指定风格：
```
Skill: frontend-design
要求：
- 风格: brutally minimal
- 内容: 基于 Markdown 报告映射
- 包含: 参数表格、对比图表、选择建议
```

## 技能调用链

本技能可以调用以下技能：

| 场景 | 调用技能 | 目的 |
|------|---------|------|
| 需要用户反馈数据 | `ian-smart-hardware-research` | 补充用户需求和痛点分析 |
| 需要技术原理分析 | `ian-hardware-technical-analysis` | 补充整机技术架构分析 |
| 需要数据校验 | `fact-checker` | 验证关键数据准确性 |
| 需要生成HTML | `frontend-design` | 生成可视化报告 |

## 质量控制清单

```
□ 模式选择正确（单品 vs 多品牌）
□ 数据收集完整（参数、价格、评价）
□ Markdown 报告结构完整
□ fact-checker 已调用并修正错误
□ HTML 报告使用 brutally minimal 风格
□ 数据来源已标注
□ 报告日期已标注
```

## 注意事项

1. **显式调用** - 本技能仅响应显式调用，不响应关键词
2. **数据时效** - 价格和技术参数会变动，需标注调研时间
3. **来源可靠** - 优先使用官方数据和权威测评
4. **客观中立** - 避免主观偏见，用事实说话
5. **必须校验** - 所有关键数据必须经过 fact-checker 验证
